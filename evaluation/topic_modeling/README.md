# Evaluation

This folder contains the evaluation pipeline and experimental results for the ENVIO LLM Coding Assistant project.

The evaluation framework compares:

- Human qualitative coding
- Standard LLM coding
- RAG-enhanced LLM coding

The goal is to examine whether retrieval-augmented generation (RAG) improves coding consistency, thematic alignment, and agreement with human annotation in qualitative research settings.

---

# Evaluation Components

## LLM Outputs

The `llm_outputs/` folder contains raw coding outputs generated by the language model under different configurations.

These outputs include:

- Standard LLM coding
- RAG-enhanced coding
- Batch coding summaries
- JSON and CSV formatted coding results

---

## Human vs LLM Comparison

The `llm_vs_human/` folder contains comparison results between:

- Human coding
- Standard LLM coding
- RAG-enhanced LLM coding

Key evaluation files include:

- `llm_vs_human_comparison.csv`
- `rag_vs_no_rag_comparison.csv`
- `no_rag_vs_human_comparison.csv`

These files are used to analyze agreement patterns, coding overlap, and thematic consistency.

---

## Topic Modeling

The `topic_modeling/` folder contains topic extraction and thematic analysis results generated from interview transcripts and coding outputs.

Included files:

- `lda_topics.csv`
- `lda_document_topics.csv`
- `qualitative_theme_summary.txt`

These analyses provide additional qualitative interpretation of the coding outputs.

---

# Evaluation Pipeline

The evaluation workflow follows the process below:

1. Input interview transcripts
2. Generate qualitative codes using the LLM
3. Apply Retrieval-Augmented Generation (RAG)
4. Save coding outputs
5. Compare LLM outputs with human annotations
6. Generate comparison metrics and thematic summaries
7. Perform topic modeling analysis

---

# Research Goal

This evaluation framework investigates whether LLM-assisted qualitative coding can support or partially automate thematic analysis workflows in social science and public health research.

The project emphasizes:

- Reproducibility
- Human–AI comparison
- Retrieval-enhanced reasoning
- Qualitative coding consistency
- Interpretable evaluation pipelines
